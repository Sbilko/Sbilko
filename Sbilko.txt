# multi-user.target: analogous to runlevel 3
# graphical.target: analogous to runlevel 5
#
# To view current default target, run:
# systemctl get-default
#
# To set a default target, run:

#
Hence, to run only CLI you can set:

Methods to Run .apk Files Natively in Linux
To run .apk files natively on Linux, two primary methods are available: Waydroid and Anbox. Both options allow you to execute Android applications without relying on traditional emulators.

Waydroid
Waydroid creates a full Android system within a container on your Linux machine. It offers better performance and hardware acceleration compared to other methods.

Key Features of Waydroid
Full Android System: Runs a complete Android environment.
Performance: Enhanced performance due to hardware acceleration.
Installation: Requires a Wayland session for optimal operation.
Anbox
Anbox allows you to run Android apps in a Linux environment by integrating them into your desktop. It is less resource-intensive than full virtualization.

Key Features of Anbox
Integration: Android apps run alongside native Linux applications.
Setup: Requires installation of kernel modules and ADB for app management.
Manual APK Installation: You can install .apk files manually using ADB.
Comparison Table
Feature	Waydroid	Anbox
Full Android OS	Yes	No
Performance	High (with hardware support)	Moderate
Integration	Seamless with Linux desktop	Integrated with Linux apps
Setup Complexity	Moderate	Moderate
ADB Support	Yes	Yes
Conclusion
Both Waydroid and Anbox provide viable solutions for running .apk files natively on Linux. Your choice may depend on your specific needs, such as performance requirements or the level of integration desired with your Linux desktop environment."
